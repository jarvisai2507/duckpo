# 🛠️ 관리자 런북 (Admin Runbook)

> **이 문서는 관리자(협력자)의 것이다.** 대통령(토니)이 붙들 영역이 아니라, 기술 환경을 세팅·유지하는 관리자의 영역이다.
> 대통령은 사건 대응에 집중하고, 여기 적힌 배관(네트워크·커넥터·무료플랜 유지)은 관리자가 책임진다.
> 계정 위계·감사 원칙은 `CLAUDE.md`를 따른다. 이 문서에는 **비밀번호·service_role 키 등 어떤 비밀도 적지 않는다.**

관련 상수(모두 공개용):
- Supabase 프로젝트: `rspxgsytxhlsauqohdbe` — `https://rspxgsytxhlsauqohdbe.supabase.co`
- publishable(anon) 키: `auth.js`에 공개. RLS가 실제 방어.

## 왜 이 런북이 필요한가 (고장 유형)

웹/원격 Claude 세션에서 Supabase(금고)에 닿지 못하면 **브리핑도, 기록 저장도** 안 된다. 원인은 셋 중 하나다:

| # | 고장 | 증상 | 누가 고치나 |
|---|------|------|------------|
| A | 환경 네트워크 정책이 Supabase 호스트 차단 | REST/커넥터가 `403 policy denial` | **관리자** (환경 설정) |
| B | Supabase MCP 커넥터 미부착 | 세션에 `execute_sql` 도구 없음 | **관리자** (커넥터 부착) |
| C | 무료 플랜 자동 정지(약 1주 무활동) | 모든 요청 실패/타임아웃 | 자동 예방(워크플로) + 대시보드 원클릭 복구 |

> 대통령은 A·B를 알 수도, 고칠 수도 없다. 세션에서 장애가 감지되면 비서실장이 **관리자용 조치 항목**으로 남긴다(무손실 폴백은 `CLAUDE.md`의 "🔌 커넥터/네트워크 장애 대응 프로토콜").

## 1. 일회성 환경 세팅 (근본 복구 · A·B)

Claude Code 웹 환경 설정에서 관리자가 한 번 잡아두면 이후 세션은 정상 작동한다.
문서: https://code.claude.com/docs/en/claude-code-on-the-web (환경 · 네트워크 정책 · 커넥터)

1. **네트워크 정책 (고장 A)**: 이 저장소의 환경이 아웃바운드 HTTPS로
   `rspxgsytxhlsauqohdbe.supabase.co` (443)에 나갈 수 있어야 한다.
   - 환경 생성/편집 화면에서 네트워크 정책을 도메인 허용이 가능한 것으로 두고,
     위 호스트를 허용 목록에 추가한다. (전면 차단 정책이면 Supabase에 못 나간다.)
2. **Supabase MCP 커넥터 (고장 B)**: 세션에서 `execute_sql`을 쓸 수 있도록
   Supabase 커넥터(MCP)를 환경/세션에 부착한다.

### 검증 (세팅 후 새 세션에서)
- 프록시 정책 확인: `curl -sS "$HTTPS_PROXY/__agentproxy/status"` →
  `recentRelayFailures`에 supabase 403이 **더 이상 안 찍히면** 통과.
- 조회 확인(커넥터): `select id, title from public.posts order by id desc limit 3;` 가 돌면 통과.
- 둘 다 되면 자동 브리핑·기록이 라이브로 작동한다.

## 2. 무료 플랜 자동 정지 예방·복구 (고장 C)

- **자동 예방**: `.github/workflows/keepalive-supabase.yml` 이 매일 공개 RPC를 1회 호출해
  DB 활동을 만들어 자동 정지를 막는다. (GitHub 러너는 네트워크 제한 없음.)
- **상태 확인**: GitHub → Actions → "Keep Supabase alive" 가 **초록**이면 정상.
  빨간 X면 프로젝트 정지 또는 설정 변경 신호다.
- **정지됐을 때 복구**: Supabase 대시보드에서 프로젝트 **Restore(재개)** 원클릭. 데이터는 보존된다.
- **주의(GitHub 정책)**: 저장소가 60일간 커밋 등 활동이 전혀 없으면 GitHub가 예약 워크플로를
  비활성화할 수 있다. 가끔 커밋이 있으면 유지되며, 필요 시 Actions에서 워크플로를 수동 실행(Run workflow)해 되살린다.

## 3. 관리자 세션 수칙 (권한·감사)

- 관리자는 **협력자**다. 세팅끝/세팅시작 전환, 원본자료 구조 변경, 기록 삭제 등
  **대통령 전용 권한은 대행하지 않는다** (`CLAUDE.md` 참조).
- 관리자 세션의 모든 조치(DB/설정/코드 변경)는 **감사실장** 명의로 게시판에 기록해 대통령에게 보고한다
  (`dept='감사실장'`, `tags`에 `'관리자활동'`). 생략 금지.
- 어떤 비밀번호·비밀 키도 게시판·문서·커밋에 남기지 않는다.
